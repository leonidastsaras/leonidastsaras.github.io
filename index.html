git/github help

git

Ορισμός ταυτότητας

Για τον ορισμό (global) ταυτότητας εκτελούμε την εντολή:

git config --global user.name "John Doe"
git config --global user.email johndoe@example.com

Για τον έλεγχο του (global) user.name εκτελούμε την εντολή:

git config --global user.name

Για τον έλεγχο του (global) user.email εκτελούμε την εντολή:

git config --global user.email

Για τη διαγραφή του global user.name και user.email

git config --global --unset user.name
git config --global --unset user.email

ή

git config --global --unset-all user.name
git config --global --unset-all user.email

Για τη διαγραφή του global user από τα Windows, επιλέξτε Credential Manager και διαγράψτε την εγγραφή από τα Generic Credentials. Σημείωση: Αναζητήστε "Credentials".

Αντίστοιχα για (local) user.name και user.email οι εντολές είναι:

git config user.name "John Doe"
git config user.email johndoe@example.com

git config user.name
git config user.email

git config --unset-all user.name
git config --unset-all user.email

Αρχικοποίηση αποθετηρίου (init)

Για την αρχικοποίηση αποθετηρίου σε υπάρχοντα κατάλογο εκτελούμε την εντολή:

git init

Έλεγχος των ρυθμίσεών μας

Για έλεγχο ρυθμίσεων, εκτελούμε την εντολή:

git config --list

Επίσης για έλεγχο user.name ή user.email

git config user.name
git config user.email

Έλεγχος της κατάστασης των αρχείων (status)

Για τον έλεγχο της κατάστασης των αρχείων εκτελούμε:

git status

Για πιο σύντομη περιγραφή:

git status -s

Λίστα αρχείων (ls-files)

Για εμφάνιση λίστας αρχείων εκτελούμε:

git ls-files

Για εμφάνιση λίστας αρχείων staged εκτελούμε:

git ls-files --stage

ή πιο σύντομα:

git ls-files -s

Για περισσότερες επιλογές: https://git-scm.com/docs/git-ls-files

Καταχώρηση τροποποιημένων αρχείων

Για την καταχώρηση αρχείων στο αποθετήριο εκτελούμε την εντολή:

git add home.html

Η καταχώρηση αφορά Modified ή/και Untracked αρχεία.

Για ταυτόχρονη καταχώρηση πολλών αρχείων (new, modified, και deleted):

git add --all

Υποβολή αλλαγών (commit)

Για την υποβολή αλλαγών εκτελούμε την εντολή:

git commit -m "message here"

Πρέπει να προστεθεί σχετικό μήνυμα με την εντολή commit.

Πρέπει επίσης να έχει δηλωθεί user.name και user.email

Υποβολή αλλαγών (commit) με amend

Σε ένα πρόσφατο commit και αφού έχετε κάνει κάποιες μικροαλλαγές μπορείτε να κάνετε:

git commit --amend

Έτσι, δεν δημιουργείτε ένα νέο commit αλλά προσθέτετε κάτι ακόμα στο τελευταίο

Μπορείτε να προσθέσετε νέο μήνυμα ή:

git commit --amend --no-edit

για να κρατήσετε το υπάρχον.

Ιστορικό υποβολών (commit) (log)

Για την εμφάνιση του ιστορικού υποβολών εκτελούμε την εντολή:

git log

Για την εμφάνιση τελευταίων υποβολών έστω 3, εκτελούμε την εντολή:

git log -3

Επαναφορά προηγούμενης υποβολής (commit)

Αν θέλουμε να επαναφέρουμε μια προηγούμενη υποβολή, εκτελούμε αρχικά:

git log

για να εμφανίσουμε τη λίστα με τις υποβολές και στη συνέχεια εκτελούμε:

git reset --hard 09d7fc58

Όπου 09d7fc58 ο κωδικός του commit.

Θα διαγραφούν όλες οι αλλαγές που έχου γίνει μέχρι τότε.

github

Κλωνοποίηση αποθετηρίου (clone)

Για την κλωνοποίηση αποθετηρίου εκτελούμε την εντολή:

git clone https://github.com/username/reponame

Δημιουργείται ο φάκελος reponame

Δεν χρειάζεται αρχικοποίηση (δημιουργείται το σχετικό .git)

Για κλωνοποίηση αποθετηρίου σε νέο φάκελο: 

git clone https://github.com/username/reponame newfolder

Ενημέρωση τοπικού αποθετηρίου από απομακρυσμένο (pull)

Για την ενημέρωση τοπικού αποθετηρίου από απομακρυσμένο εκτελούμε την εντολή:

git pull

Η pull είναι συνδυασμός των fetch και merge

Ώθηση δεδομένων σε απομακρυσμένα αποθετήρια (push)

Όταν έχουμε κάποιο έργο στο οποίο έχουμε κάνει αλλαγές και το έχουμε φτάσει σε σημείο που θέλουμε να το κοινοποιήσουμε, θα πρέπει να το ωθήσουμε και εκτελούμε την εντολή:

git push

Η εντολή αυτή θα εκτελεστεί επιτυχώς μόνο αν έχουμε κλωνοποιήσει από έναν διακομιστή στον οποίο έχουμε δικαιώματα εγγραφής δεδομένων και αν κανείς άλλος δεν έχει ωθήσει δεδομένα στο μεσοδιάστημα

Δημιουργία νέου αποθετηρίου για ώθηση νέου git

Αν έχουμε δημιουργήσει ένα νέο git και θέλουμε να το ανεβάσουμε στο github σε ένα νέο repo, πρέπει:

  • να δημιουργήσουμε στο github νέο αποθετήριο
  • να εκτελέσουμε τις παρακάτω εντολές: 
    git remote add origin https://github.com/user-name/repo-name.git
git branch -M main
git push -u origin main

    όπου user-name και repo-name θα αντικατασταθούν από τις αντίστοιχες τιμές.

Θα πρέπει να έχουν προηγηθεί οι παρακάτω εντολές όταν πρόκειται για ένα νέο project:

git init
git add README.md
git commit -m "some message"

Branches

Για την εμφάνιση των branches εκτελούμε την εντολή:

git branch

ή

git branch --list

Για την δημιουργία ενός νέου branche, έστω test, εκτελούμε την εντολή:

git branch test

Για αλλαγή σε άλλο branche, έστω test, εκτελούμε την εντολή:

git switch test
git checkout test

Για μετονομασία ενός branche από έστω test σε tester, εκτελούμε την εντολή:

git branch -m test tester

Για τη διαγραφή ενός branche από έστω test, εκτελούμε την εντολή:

git branch -D test

Για το merge ενός branche έστω test στο main, αλλάζουμε στο main και εκτελούμε την εντολή:

git merge test
